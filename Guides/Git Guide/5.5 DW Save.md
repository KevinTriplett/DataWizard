# 5.5 DW Save

DW Save is your intentional save point -- one keystroke to commit your work and push it to GitHub. Think of it like Cmd+S in any app, except it also backs up your work and shares it with collaborators.

- **Mac:** Cmd+Shift+S
- **Windows:** Ctrl+Shift+S

## Why Manual Save + Auto-Backup

DW uses a two-layer approach to keeping your work safe:

**DW Save (manual)** is your primary workflow. You hit it at natural breakpoints -- when you finish a work session, before a call, before stepping away, or when you want a collaborator to see your latest changes. You control when your work goes out.

**Auto-backup (scheduled)** is your safety net. It runs on a longer cycle in the background, catching anything you forgot to manually save. It's deliberately not set to run every minute -- you don't want half-finished notes and mid-thought edits pushing to GitHub automatically. The auto-backup is there for peace of mind, not as your main workflow.

## Single Project Setup

This is the default for most users. It uses the Obsidian Git plugin you already installed in Section 3.0 -- no extra plugins or scripts needed.

### Set up the hotkey

1. In Obsidian: **Settings > Hotkeys**
2. Search for **Obsidian Git: Commit-and-sync**
3. Click the **+** icon, then press **Cmd+Shift+S** (Mac) or **Ctrl+Shift+S** (Windows)

### Test it

1. Create a test note called `_test-dw-save.md` with any content
2. Press **Cmd+Shift+S**
3. You should see a notification: "Committed X files and pushed X files to remote" (or "Nothing to commit" if it was already synced)
4. Delete the test note and press **Cmd+Shift+S** again to push the deletion

If nothing happens, see Troubleshooting below.

### Set your backup schedule

In Obsidian: **Settings > Community Plugins > Git** (gear icon).

The key setting is **Auto commit-and-sync interval (minutes)**. This is how often the safety net runs. Some guidance:

- **15-30 minutes** -- good default for collaborative projects. Long enough that you're not pushing every stray edit, short enough that you won't lose much if you forget to manually save.
- **60 minutes or more** -- fine for solo vaults or projects where you're the only editor. You're relying on DW Save as your primary workflow anyway.
- **1-5 minutes** -- only if you specifically want aggressive auto-sync (e.g., you're doing rapid back-and-forth with a collaborator and want near-real-time). Be aware this will push incomplete work.

Pick what fits your workflow. You can always change it later.

## Multi-Project Setup

This is for vaults that contain multiple separate git repositories -- for example, if your Obsidian vault has a shared project folder that you and a collaborator sync through GitHub, and that folder is its own git repo living inside your larger vault. You might have several of these: one for each collaborative project, plus the DW Seed itself.

The Single Project approach only syncs one repo at a time. Multi-Project setup lets you sync all of them with a single Cmd+Shift+S by using a script that loops through each repo and pushes them all.

### What you need

1. **Shell Commands plugin** -- an Obsidian community plugin that lets you run terminal commands from inside Obsidian
2. **datawizard-sync.sh** -- the sync script (included in `Seed/Scripts/`)
3. **~/.datawizard-sync.conf** -- a config file listing which repos to sync

### Setup

Follow the full walkthrough in `Seed/Scripts/datawizard-sync-setup.md`. It covers installing the plugin, copying the script, creating the config, setting up the hotkey, and optionally adding a scheduled safety net via launchd.

### Set your backup schedule

The Multi-Project safety net uses macOS launchd (a background scheduler) instead of the Obsidian Git plugin. The default is every 2 hours. Same principle as above -- it's a safety net, not your primary workflow. If you want to change the interval, edit the `StartInterval` value in the launchd plist (measured in seconds; 7200 = 2 hours). See `datawizard-sync-setup.md` Step 5 for details.

## Troubleshooting

**DW Save does nothing when I press the hotkey.**
The most common cause: your plugin got disabled. Hard restarts and Obsidian updates can toggle community plugins off without warning. Go to **Settings > Community Plugins** and check that the relevant plugin is enabled:
- Single Project: look for **Git** (by Vinzent03)
- Multi-Project: look for **Shell commands**

If the plugin is enabled but the hotkey still doesn't work, check **Settings > Hotkeys** and search for the command to verify the binding is still there.

**"Git is not ready" error.**
The Obsidian Git plugin can't find the git binary. See Section 3.0 for the fix (setting the custom git binary path).

**"Sync conflict" or merge conflict.**
Two people edited the same file between syncs. See Section 5.0 for how to resolve merge conflicts.
