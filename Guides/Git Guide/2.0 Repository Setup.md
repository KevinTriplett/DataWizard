# 2.0 Repository Setup

How to create a GitHub repository and connect your vault or project folder to it.

## Create the GitHub Repository

1. Go to https://github.com/new
2. Name the repository (e.g., your vault name or project name, no spaces)
3. Set visibility to **Private** (unless you intend to share publicly)
4. **Do NOT** check "Add a README," "Add .gitignore," or "Choose a license" -- the repo must start empty
5. Click **Create repository**
6. Copy the repo URL -- it looks like `https://github.com/username/repo-name.git`

## Create a .gitignore

Before initializing git, create a `.gitignore` file in your vault root to exclude files that shouldn't be tracked. Create a plain text file named `.gitignore` (note the leading dot) with this content:

```
# Obsidian ephemeral state
.obsidian/workspace.json
.obsidian/workspace-mobile.json
.obsidian/app.json

# AI plugin index files (can exceed GitHub's 100MB limit)
.obsidian/copilot-index-*.json
.obsidian/smart-connections/
.smart-env/
.copilot/

# System files
.DS_Store
Thumbs.db
.trash/

# Lock files
.~lock.*
```

Customize as needed. If your vault has other large generated files (vector indexes, embedding caches), add them here. GitHub rejects individual files over 100MB.

## Initialize and Push

```bash
cd "/path/to/your/vault"
git init
git add .
git commit -m "initial vault backup"
git remote add origin https://github.com/username/repo-name.git
git branch -M main
git push -u origin main
```

Replace the path and URL with your actual values. The initial push may take a few minutes for large vaults.

Visit your repo URL on GitHub -- you should see your vault files.

## SSH Keys (Recommended)

SSH keys eliminate password prompts on every push and pull. If you'll be using git regularly, set this up now.

**Generate a key:**

```bash
ssh-keygen -t ed25519 -C "your-email@example.com"
```

Press Enter for the default file location. Passphrase is optional.

**Add the public key to GitHub:**

Mac:
```bash
cat ~/.ssh/id_ed25519.pub
```

Windows (PowerShell):
```powershell
Get-Content "$env:USERPROFILE\.ssh\id_ed25519.pub"
```

Copy the output, then go to GitHub > Settings > SSH and GPG Keys > New SSH Key > paste > Add.

**Test it:**

```bash
ssh -T git@github.com
```

You should see "Hi [username]! You've successfully authenticated."

**Switch your repo to SSH (if you initially set up with HTTPS):**

```bash
cd "/path/to/your/vault"
git remote set-url origin git@github.com:username/repo-name.git
```

## Nested Repos (for DW Projects Inside a Vault)

If you're setting up a git repo for a project folder that lives inside a vault that already has its own git repo, you need to isolate them. Git gets confused by a repo inside a repo unless you explicitly separate them.

After initializing git inside the project folder:

1. Add the project path to the parent vault's `.gitignore`:
   ```bash
   echo "_ProjectFolder/" >> "/path/to/vault/.gitignore"
   ```

2. Remove the project from the parent's tracking:
   ```bash
   cd "/path/to/vault"
   git rm -r --cached "_ProjectFolder/"
   git commit -m "Stop tracking _ProjectFolder (now a nested repo)"
   ```

Both steps are required -- the gitignore prevents future tracking, the `rm --cached` removes existing tracking.

For the full nested repo setup pattern and a registry of existing repos, see the 0.6 Registry (Git Repositories section) in any DataWizard project.
