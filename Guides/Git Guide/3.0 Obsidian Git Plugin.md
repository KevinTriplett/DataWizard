# 3.0 Obsidian Git Plugin

The Obsidian Git plugin automates commits and pushes from within Obsidian so you rarely need the terminal after initial setup.

## Install

1. In Obsidian: **Settings > Community Plugins**
2. Turn on community plugins if prompted
3. Click **Browse**, search for **Git** (the one by **Vinzent03**)
4. Click **Install**, then **Enable**

## Configure

Open Settings > Community Plugins > Git (click the gear icon).

### Fix the git binary path (Mac -- often required)

Obsidian may not find the Homebrew git installation. Get the path:

```bash
which git
```

In Obsidian Git settings, scroll to **Advanced** > **Custom Git binary path** > enter the full path (e.g., `/opt/homebrew/bin/git`).

Click **Reload**. If "Git is not ready" persists, fully quit and relaunch Obsidian.

Windows users generally don't need this step -- the Git for Windows installer adds git to the system PATH.

### Auto-sync settings

See Section 5.5 (DW Save) for guidance on choosing your auto-sync interval and how it relates to manual saves.

| Setting | Value | Notes |
|---|---|---|
| Auto commit-and-sync interval (minutes) | `15` | Safety net interval. See Section 5.5 for guidance |
| Auto commit-and-sync after stopping file edits | On | Commits when you stop typing |
| Auto commit-and-sync only staged files | Off | Commit all changes |
| Push on commit-and-sync | On | Push to GitHub on every commit |
| Pull on commit-and-sync | On | Pull before pushing |
| Pull on startup | On | Sync when Obsidian opens |
| Hide notifications for no changes | On | Suppresses noisy "nothing to commit" popups |

### Commit messages

| Setting | Value |
|---|---|
| Commit message on auto commit-and-sync | `auto: {{date}} on {{hostname}}` |
| List filenames affected in commit body | On |

### Commit author

| Setting | Value |
|---|---|
| Author name for commit | Your name |
| Author email for commit | Your GitHub email |

## DW Save (Keyboard Shortcut)

The hotkey setup and testing for DW Save is covered in **Section 5.5 DW Save**. Complete the plugin configuration above first, then follow Section 5.5 to set up your save hotkey and choose your backup schedule.

## Obsidian File Recovery (Secondary Safety Net)

This is a built-in Obsidian core plugin -- no installation needed. It provides quick in-app snapshots for individual file recovery without touching the terminal.

1. Settings > Core Plugins > **File Recovery** > Enable (if not already on)
2. Click the gear icon next to File Recovery
3. Set **Snapshot interval** to `5` minutes
4. Set **History length** to `30` days (up from the 7-day default)

To recover a file: open the Command Palette, search "File Recovery: View snapshots for current note."
