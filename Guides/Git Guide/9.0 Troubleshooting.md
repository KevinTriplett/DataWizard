# 9.0 Troubleshooting

Common issues and their fixes.

## Push Rejected: File Exceeds 100MB

GitHub rejects individual files over 100MB. This usually happens with AI plugin index files (Copilot, Smart Connections) or large PDFs.

**Step 1:** Note the filename from the error message.

**Step 2:** Add the file pattern to `.gitignore`:

```
# Example: exclude Copilot index
.obsidian/copilot-index-*.json
```

**Step 3:** The file is already in commit history, so the .gitignore alone won't help. Rewrite history to remove it:

```bash
git stash                           # save any uncommitted changes
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch "PATH/TO/LARGE/FILE"' \
  --prune-empty --tag-name-filter cat -- --all
git stash pop                       # restore uncommitted changes
git push -u origin main --force
```

If `git stash pop` fails with a "not a stash-like commit" error, the stash was rewritten along with history. This is harmless -- the stashed changes were likely minor (cursor positions, etc.). Continue without it.

## Authentication Failure on Push

GitHub no longer accepts password authentication for HTTPS.

**Option A -- Personal Access Token:**

1. GitHub > Settings > Developer settings > Personal access tokens > Tokens (classic)
2. Generate new token with `repo` scope
3. When git prompts for a password, enter the token instead

**Option B -- Switch to SSH (recommended):**

See Section 2 (SSH Keys). This is the permanent fix.

## "Git is not ready" in Obsidian Git Settings

1. Run `which git` in terminal -- note the full path
2. Obsidian Git settings > Advanced > Custom Git binary path > enter the full path
3. Click Reload
4. If it persists, fully quit and relaunch Obsidian

## Obsidian Git Not Auto-Committing

Check that:

- Auto commit-and-sync interval is set to `1` or `5` (not `0`)
- The plugin is enabled (Settings > Community Plugins)
- "Git is not ready" is NOT showing in settings
- There are actually uncommitted changes (`git status` in terminal)

## Merge Conflict on Auto-Pull

The plugin pulled changes that conflict with your local edits. The conflicted file will contain `<<<<<<<` markers.

1. Open the file in Obsidian
2. Look for the conflict markers
3. Keep the version you want, delete the markers
4. Save
5. Next auto-sync commits the resolution

If you see many conflicts at once (e.g., after being offline for a while), consider resolving them one at a time from the terminal:

```bash
git status                          # see which files are conflicted
# edit each file to resolve
git add "path/to/resolved-file.md"
git commit -m "resolve merge conflicts"
```

## zsh History Expansion Error (Mac)

If your vault path contains `!` (e.g., `_!nbox/`), zsh interprets it as a history expansion character. Wrap paths in single quotes:

```bash
cd '/Users/you/Vaults/Regen Vault/_!nbox/'
```

Or disable history expansion: `set +o histexpand`

## "fatal: not a git repository"

You're not in the right directory. Make sure you `cd` into the correct folder:

- For vault operations: `cd "/path/to/vault"`
- For Seed operations: `cd "/path/to/vault/_DataWizard/Seed"`
- For project operations: `cd "/path/to/vault/_ProjectFolder"`

Nested repos have their own `.git` -- commands must be run from inside the correct repo.

## Push Succeeds but Collaborator Doesn't See Changes

1. Check that the collaborator's Obsidian Git plugin is running (status bar should show recent activity)
2. Have them manually pull: `Cmd+P` > `Git: Pull`
3. If still not showing: check they're on the same branch (`git branch` in terminal -- the starred branch is the current one)
4. Verify the push actually reached GitHub: check the repo's commit history on github.com

## "HEAD detached" Warning

This means you're looking at a specific commit rather than being on a branch. Common after running `git checkout <hash>` to inspect old state.

To get back to normal:

```bash
git checkout main
```

## Obsidian Shows Duplicate or Ghost Files After Git Operations

Sometimes Obsidian's file index gets out of sync with what git did (especially after branch switches or resets). Fix:

1. Close and reopen Obsidian (forces a re-index)
2. If duplicates persist, check `git status` -- the "ghost" files may be untracked leftovers. Delete them manually or `git clean -fd` (caution: this deletes all untracked files).
