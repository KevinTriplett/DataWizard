# 5.0 Daily Workflow

How sync works day to day, whether you're working solo or collaborating.

## DW Save and Auto-Backup

Your primary sync tool is **DW Save** (Cmd+Shift+S / Ctrl+Shift+S) -- see **Section 5.5** for full setup and how to choose your backup schedule. Auto-backup runs in the background as a safety net on a longer cycle.

Between DW Save and auto-backup, sync flows like this:

```
You finish a work session or reach a breakpoint
  -> You hit DW Save (Cmd+Shift+S)
    -> Your work commits and pushes to GitHub
      -> Collaborator's plugin auto-pulls
        -> They see your changes

Between manual saves, auto-backup catches anything you missed
  -> Runs on your chosen interval (e.g., every 15-30 min)
    -> Same commit-and-push, just automatic
```

## Check Sync Status

Look at the bottom status bar in Obsidian -- the Git plugin shows last commit time and pending changes.

From terminal:

```bash
cd "/path/to/your/vault"
git log --oneline -5    # last 5 commits
git status              # uncommitted changes
```

## Merge Conflicts

These happen when two people edit the same lines of the same file between syncs. In practice they're rare -- collaborators are usually in different files.

When a conflict occurs, git marks the file with both versions:

```
<<<<<<< HEAD
Your version of the text
=======
The other person's version of the text
>>>>>>> origin/main
```

To resolve:

1. Open the file in Obsidian (or any text editor)
2. Keep the version you want (or combine both)
3. Delete the `<<<<<<<`, `=======`, and `>>>>>>>` markers
4. Save the file
5. Next auto-sync commits the resolution

If you're on a call with someone, just say "I'm editing X" to avoid stepping on each other.

## Before Bulk Operations

Before running any pipeline, script, or batch process that will modify many files, run a manual commit-and-sync first:

`Cmd+P` > `Git: Commit-and-sync`

This creates a clean snapshot you can roll back to if the operation does something unexpected. See Section 7 for recovery procedures.

## Recommended Habits

- **Push before stepping away.** Hit Cmd+Shift+S before calls, breaks, or ending your work session.
- **Pull before starting.** The plugin pulls on startup, but if Obsidian has been open a while, do a manual sync before starting a focused work session.
- **Communicate edits.** If you're working at the same time as someone else, a quick "I'm editing the Codex" prevents conflicts.
- **Commit messages matter.** The auto-commit messages include filenames, which helps when scanning history. For manual commits (via terminal), write a short description of what changed and why.
